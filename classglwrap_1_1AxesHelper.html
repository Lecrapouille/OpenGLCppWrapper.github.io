◆ doUpdate()

virtual void glwrap::Node3D::doUpdate ( float const  )
inlinevirtualinherited
Note
: the project https://github.com/vahlers/scg3 traverse the tree using the vistor design pattern. I did not follow because I am a big fan of visitors: in my personal opinion they make the code harder to understand. In the course https://research.ncl.ac.uk/game/mastersdegree/ graphicsforgames/scenegraphs/Tutorial%206%20-%20Scene%20Graphs.pdf they let derived classes override the method update() but suppose the developper will call the method update() of the parent class. I think this is an error prone pratice. I think doUpdate() does the same thing in a more simpler way for the developper.
Parameters
dtdelta time from the previous update.
