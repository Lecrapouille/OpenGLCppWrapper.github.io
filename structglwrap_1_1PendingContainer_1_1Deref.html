Detailed Description

template<class T>
struct glwrap::PendingContainer< T >::Deref

operator[] when not const will always be used as a setter even when it is supposed to call a getter. For example a = std::container[0] will call the std::container::set() method while desired to call the get() method (in the same way than (PendingContainer[0] = a).

For PendingContainer this is not a desirable behavior because this will set the nth element as dirty (pending data) and imply a flush to the GPU memory in the case of VBO.

To allow C++ to understand to correctly select setter getter to use this class is needed.
